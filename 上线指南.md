# NovRow 上线指南 — 真实注册 + Google 登录 + 部署

跟着下面的步骤一步一步做。需要的账号都是**免费**的：GitHub、Supabase、Google Cloud、Vercel。

整个流程分 8 步。**你负责注册账号和点击设置**，**代码部分我来帮你改**（在第 6 步）。

---

## 第 1 步：把代码放上 GitHub（为了之后自动部署）

1. 如果还没装 Git：到 https://git-scm.com/download/win 下载安装（一路 Next）。
2. 到 https://github.com 注册/登录，点右上角 **+ → New repository**。
   - Repository name 填 `novrow`，选 **Private**，**不要**勾任何初始化选项，点 **Create repository**。
3. 在 `C:\boatProject` 文件夹地址栏输入 `cmd` 回车，打开命令行，依次运行：
   ```
   git init
   git add .
   git commit -m "NovRow app"
   git branch -M main
   git remote add origin https://github.com/你的用户名/novrow.git
   git push -u origin main
   ```
   （把 `你的用户名` 换成你的 GitHub 用户名。第一次 push 会让你登录 GitHub。）

> 备注：`.gitignore` 已经帮你排除了 `node_modules`，不用管它。

## 第 2 步：创建 Supabase 项目（负责登录 + 存用户资料）

1. 到 https://supabase.com ，用 GitHub 账号登录。
2. 点 **New project**：
   - Name：`novrow`
   - Database Password：自己设一个并**记下来**
   - Region：选离你近的（如 Singapore）
   - 点 **Create new project**，等 1–2 分钟。
3. 左侧菜单 **Settings（齿轮）→ API**，记下两样东西（之后给我）：
   - **Project URL**（形如 `https://xxxxx.supabase.co`）
   - **anon public** key（很长一串。这是公开 key，放在前端是安全的）

## 第 3 步：建数据库表（存用户名、邮箱、闯关进度、徽章）

1. 左侧菜单 **SQL Editor → New query**，把下面整段粘贴进去，点 **Run**：

```sql
-- 用户资料表：每个用户一行，存用户名/全名/闯关进度
create table public.profiles (
  id uuid references auth.users on delete cascade primary key,
  username text,
  full_name text,
  progress jsonb default '{}'::jsonb,
  created_at timestamptz default now()
);

-- 打开行级安全：每个人只能看/改自己的数据
alter table public.profiles enable row level security;

create policy "read own profile"  on public.profiles for select using (auth.uid() = id);
create policy "update own profile" on public.profiles for update using (auth.uid() = id);
create policy "insert own profile" on public.profiles for insert with check (auth.uid() = id);

-- 新用户注册时，自动建一行 profile
create function public.handle_new_user()
returns trigger language plpgsql security definer as $$
begin
  insert into public.profiles (id, username, full_name)
  values (new.id, new.raw_user_meta_data->>'username', new.raw_user_meta_data->>'full_name');
  return new;
end;
$$;

create trigger on_auth_user_created
  after insert on auth.users
  for each row execute procedure public.handle_new_user();
```

看到 “Success. No rows returned” 就对了。

## 第 4 步：开启邮箱注册

1. 左侧 **Authentication → Providers → Email**，确认是 **Enabled**（默认就是开的）。
2. （可选）**Authentication → Sign In / Providers** 里可以关掉 “Confirm email”，这样测试时注册完不用点确认邮件就能直接登录。正式上线建议保留确认邮件。

---

## 第 5 步：设置 Google 登录

**5a. 在 Google Cloud 拿到 OAuth 凭据**
1. 到 https://console.cloud.google.com ，登录后上方点 **新建项目**，名字 `novrow`，创建并选中它。
2. 左侧 **API 和服务 → OAuth 同意屏幕**：User Type 选 **外部 / External**，填应用名 `NovRow`、你的邮箱，其他默认，一路保存。
3. 左侧 **API 和服务 → 凭据 → 创建凭据 → OAuth 客户端 ID**：
   - 应用类型：**Web 应用**
   - **已获授权的重定向 URI** 添加这一条（把 xxxxx 换成你第 2 步的项目地址）：
     ```
     https://xxxxx.supabase.co/auth/v1/callback
     ```
   - 创建后会得到 **客户端 ID** 和 **客户端密钥**，复制下来。

**5b. 填进 Supabase**
1. Supabase 左侧 **Authentication → Providers → Google**，打开 Enabled。
2. 把上一步的 **Client ID** 和 **Client Secret** 粘进去，保存。

**5c. 设置允许的网址**
1. Supabase **Authentication → URL Configuration**：
   - Site URL 先填 `http://localhost:5173`（本地测试用，部署后再改成正式网址）。
   - Redirect URLs 也加上 `http://localhost:5173`。

## 第 7 步：部署到 Vercel（让网站上线）

1. 到 https://vercel.com ，用 GitHub 登录。
2. 点 **Add New → Project**，选你的 `novrow` 仓库，**Import**。
3. Framework 会自动识别为 **Vite**，不用改。
4. 展开 **Environment Variables**，加两条（值就是第 2 步的）：
   - `VITE_SUPABASE_URL` = 你的 Project URL
   - `VITE_SUPABASE_ANON_KEY` = 你的 anon key
5. 点 **Deploy**，等 1–2 分钟，会给你一个网址，例如 `https://novrow.vercel.app`。

---

## 第 8 步：上线后的收尾

1. 回到 Supabase **Authentication → URL Configuration**：
   - 把 Site URL 改成你的 Vercel 网址，Redirect URLs 也加上它。
2. 回到 Google Cloud 的 OAuth 凭据，在“已获授权的来源/重定向”里也加上你的 Vercel 网址。
3. 用手机/电脑打开 Vercel 网址，注册一个真账号、试试 Google 登录、闯一关，看看徽章是否保存。

以后只要 `git push`，Vercel 会自动重新部署。
